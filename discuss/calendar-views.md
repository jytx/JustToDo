# 日历视图方案（周 / 月 / 年）

> 参考：滴答清单官方帮助文档
> - 周视图：<https://help.dida365.com/articles/6950641140068515840>
> - 月视图：<https://help.dida365.com/articles/6950640298334617600>
> - 年视图：<https://help.dida365.com/articles/7397517843383713792>
>
> 状态：**方案评审中**（不进入实施；用户可能后续指定日历组件库）

---

## 一、目标

参照滴答清单的"日历模块"，在 JustToDo 任务区（`/today`、`/upcoming`、`/all` 三个智能视图）顶部 topbar 增加一个**视图切换器**下拉，支持：

| 视图 | 路由 | 简述 |
|------|------|------|
| 周视图 | `/week` | 时间轴（左侧小时轴 + 右侧 7 列）+ 全天任务栏 |
| 月视图 | `/month` | 7 列 × 5~6 行网格，每格一个日期 + 任务块 |
| 年视图 | `/year` | 4 × 3 网格的迷你月日历 + 可选热力图 |

切换器默认显示"列表"（即当前的 3 个智能视图保持原样），下拉可选 **周/月/年**。选完后**整个主区域替换**为对应视图（不再是任务列表）。

## 二、整体架构

### 2.1 路由规划

| 路由 | 名称 | 视图组件 |
|------|------|----------|
| `/today` | today | `SmartView.vue`（不变） |
| `/upcoming` | upcoming | `SmartView.vue`（不变） |
| `/all` | all | `SmartView.vue`（不变） |
| `/week` | week | **`WeekView.vue`**（新建） |
| `/month` | month | **`MonthView.vue`**（新建） |
| `/year` | year | **`YearView.vue`**（新建） |

> 原 3 个智能视图路由完全不动，避免破坏现有 `TheSidebar` 跳转、徽标计数、键盘导航等。
> 视图切换器只是 topbar 的一个下拉菜单，点选项 → `router.push`。

### 2.2 与 TheSidebar 协调

- TheSidebar 现在已有"今天 / 未来 7 天 / 全部"3 个智能视图入口
- **不**在 TheSidebar 加"周/月/年"入口——保持 TheSidebar 是任务视图导航，日历视图只走 topbar
- 周/月/年路由下 TheSidebar 仍显示（任务二级导航），符合 `showTaskSidebar` 现有逻辑（`route.name` 是 today/upcoming/all/list/tag 才显示——**这点需要放宽**）

> ⚠️ **修正**：`showTaskSidebar` 现在排除 week/month/year。需要改成包含它们，否则切到周视图时侧栏消失，视觉上很奇怪（顶部还显示着排序/搜索按钮但侧栏空了）。

### 2.3 数据层

- **复用现有后端**：任务模型已经有 `due_start_at` / `due_end_at`，足以支撑按月份、按周、按年查询
- **新增 1 个后端命令**：`tasks_in_range(start_ts, end_ts) -> Task[]`（或复用现有 `db.getTasksByList` + 客户端过滤；先按客户端过滤方案来，省得改后端）
- **复用现有 store**：`useTaskStore`，视图层调用 `taskStore.refreshCounts()` / `loadTasks(...)`；如需跨视图缓存，新建 `useCalendarStore`（保存当前 focus 日期、热力图模式等 UI 状态）

### 2.4 视图切换器 UI

参考滴答清单截图：topbar 右侧，搜索/新建/主题按钮旁边，加一个**带下拉的按钮**：

```
┌─────────────────────────────────────────────────────┐
│  [🔍] [+] [☀/🌙] [📅 周 ▾]  [↕ 排序]                │
└─────────────────────────────────────────────────────┘
```

按钮内文字随当前视图动态变化（"周 / 月 / 年 / 列表"），下拉打开后是 4 个选项（**不**复刻滴答清单的 7 个——日程/多日/多周超出本期范围）：

```
┌──────────┐
│  列表    │ L  （保留现状）
│  周      │ W
│  月      │ M
│  年      │ Y
└──────────┘
```

快捷键 `L/W/M/Y` 切换（仿滴答 `D/W/M/Y` 但用 L 代替 D）。

## 三、三个视图的设计

### 3.1 月视图（`MonthView.vue`）—— 主战场

**视觉**（参考截图 `2e08ee047a7d43265c44d94b829704b1.png`）：

```
┌─────────────────────────────────────────────────┐
│ [<]  2025 年 12 月  [今天]  [>]              [⋯]│  ← 顶部工具条
├──────┬──────┬──────┬──────┬──────┬──────┬──────┤
│  周一 │  周二 │  周三 │  周四 │  周五 │  周六 │  周日 │
├──────┼──────┼──────┼──────┼──────┼──────┼──────┤
│ 25  │  26  │  27  │  28  │  29  │  30  │  1   │
│      │      │      │      │      │      │      │
├──────┼──────┼──────┼──────┼──────┼──────┼──────┤
│ 2    │  3   │  4   │ ...                              │
│ ☑ 任务 │ ...                                   │
│ ☑ 任务 [+2]                                     │
└──────┴──────┴──────┴──────┴──────┴──────┴──────┘
```

- 每格宽度均分（约 130px），高度可滚动固定（行高 120px 左右）
- 单元格内容：日期数字（加粗）+ 农历（可选，先不做）+ 最多 3 个任务块 + `+N` 溢出指示
- **任务块**：左侧小方块复选框（沿用 TaskCheckbox）+ 标题，超过一行省略
- **当前日**：日期数字 + 蓝色圆形背景
- **跨月日期**：置灰
- **今日**：上方显示红/蓝横条

**交互**：
1. 点击空白 → 弹出 popover（"X 月 X 日"+"准备做什么？"+"清单名"）→ QuickAdd 风格，自动回填日期
2. 点击任务块 → 打开 TaskDetailPanel
3. 点击日期数字 → 跳到 `/week?date=...`（跳周视图）
4. 双击日期数字 → 跳回今天（快捷）
5. 顶部 ‹ › 切上/下个月，"今天" 跳回本月
6. ⋯ 菜单：显示范围筛选（按清单/标签，复用现有 list/tag filter 逻辑）
7. 拖拽任务块改日期（高优先级——滴答清单核心交互，本期可做简化版：拖到日期格子上即改 due_start）

**视觉简化**：
- 本期**不**实现热力图（属于年视图）
- 本期**不**实现阴历
- 本期**不**实现"格子样式" vs "时间轴样式"——只做月视图的格子样式

### 3.2 周视图（`WeekView.vue`）—— 时间轴版

**视觉**（参考截图 `3fb3d32b07fdb525412963b42c593331.png`）：

```
┌──────────────────────────────────────────────────────────────┐
│ [<]  2025 年 12 月  [今天]  [>]           [⋯]              │
├──────────────────────────────────────────────────────────────┤
│ 第 51 周  │ 周一  │ 周二  │ 周三  │ 周四  │ 周五  │ ...    │
├──────────────────────────────────────────────────────────────┤
│  全天任务栏  │  ☑ 任务1 │ ☑ 任务2 │ ...                     │
│  (高度可拖)  │         │         │                          │
├──────────────────────────────────────────────────────────────┤
│ 14:00 │      │       │       │       │       │       │     │
│ 15:00 │ ☑ 任 │       │ ☑ 任  │       │       │       │     │
│ 16:00 │   务 │       │   务  │       │       │       │     │
│ ...   │      │       │       │       │       │       │     │
│ 18:00 │ ☑ 任 │ ☑ 任 │       │       │       │       │     │
│ ──────── 当前时间红线 ───────────────────────────            │
│ 19:00 │      │       │       │       │       │       │     │
│ 20:00 │      │ ☑ 任 │ ☑ 任 │       │       │       │     │
└──────────────────────────────────────────────────────────────┘
```

- 左侧 50px 小时轴（仅显示工作时间段 8:00 ~ 22:00，超出范围可隐藏/滚动）
- 右侧 7 列，每天宽度均分（≈150px）
- 全天任务栏在时间轴上方独立一行（高度可拖，本期固定）
- **任务块**：时间段任务按起止时间纵向撑开（高度 ∝ 时长），含复选框 + 标题 + 时间段
- **当前时间红线**：横穿所有列，左侧标"HH:MM"

**交互**：
1. 点击时间轴空白 → 弹出 popover（"X 月 X 日 HH:MM" + QuickAdd，自动回填时间）
2. 长按拖动空白 → 选中时间段（仿滴答桌面端）
3. 点击任务块 → 打开 TaskDetailPanel
4. 拖拽任务块到新时间 → 改 due_start（本期做简化版：只改时间不改日期）
5. 顶部 ‹ › 切上/下周；"今天" 回本周；点击周数跳月视图

**视觉简化**：
- 本期**不**实现缩放、隐藏时间段、双指手势
- 本期**不**实现多日视图 / 多周视图（路由上压根不加这两个）

### 3.3 年视图（`YearView.vue`）—— 热力图复盘

**视觉**（参考截图 `27333ec992987be01b41ecb86d3dbfd4.png` + `890bdc6ca15f070fc0b83db61ca6b83e.png`）：

```
┌──────────────────────────────────────────────────────────────┐
│ [<]  2025 年  [今天]  [>]              [⋯]                  │
├──────────────────────────────────────────────────────────────┤
│  一月     │  二月     │  三月     │  四月                     │
│ 日 一 二...│ 日 一...  │ 日 一...  │ 日 一...                   │
│ 29 30 31  │ 26 27 28  │ 23 24 25  │ 30 31 1 2                 │
│ 5  6  7 8 │ 2  3  4 5 │ 2  3  4 5 │ 6  7  8 9                 │
│ ...       │ ...       │ ...       │ ...                       │
├───────────┼───────────┼───────────┼───────────────────────────┤
│  五月     │  六月     │  七月     │  八月                     │
│ ...                                                            │
├──────────────────────────────────────────────────────────────┤
│  九月     │  十月     │  十一月   │  十二月                   │
│ ...                                                            │
└──────────────────────────────────────────────────────────────┘
```

- 4 列 × 3 行网格，每个子单元是一个迷你月历（日~六表头 + 日期数字）
- 月份名可点击 → 跳 `/month?date=YYYY-MM`
- 日期数字可点击 → 跳 `/week?date=YYYY-MM-DD`
- **当前日**：蓝色圆点
- **热力图模式**（默认开启）：每个日期格子底色按当天完成的任务数上色
  - 颜色梯度：`var(--jt-surface)` → `--jt-accent-soft` → `--jt-primary`（透明度分级）
  - 关闭热力图 → 格子保持纯色（参考截图 `0df73b145c5560ff38ab1efcd8146a09.png`）
- ⋯ 菜单：切换热力图显示 / 筛选清单

**视觉简化**：
- 热力图只基于"完成任务数"（不区分优先级/清单）——简化实现
- 本期**不**实现农历、节日高亮
- 本期**不**实现"过去 5 年对比"等高级统计

## 四、复用与共享组件

### 4.1 现有可复用

| 组件 / Store | 用途 |
|--------------|------|
| `TaskCheckbox.vue` | 任务块前的复选框 |
| `TaskDetailPanel.vue` | 任务详情面板（点任务块打开） |
| `QuickAddDialog.vue` | 顶部 + 按钮触发的全局快速添加（不带日期上下文） |
| `useTaskStore` | 任务 CRUD + 计数 |
| `useListStore` / `useTagStore` | 清单/标签筛选 |
| `topbar` 布局 | 在 AppLayout 里加一个下拉按钮即可 |

### 4.2 需要新建

| 文件 | 用途 |
|------|------|
| `src/views/WeekView.vue` | 周视图主体 |
| `src/views/MonthView.vue` | 月视图主体 |
| `src/views/YearView.vue` | 年视图主体 |
| `src/components/ViewSwitcher.vue` | topbar 视图切换器下拉 |
| `src/components/CalendarPopover.vue` | 月/周视图点击空白处弹出的 QuickAdd 风格 popover |
| `src/composables/useCalendarNav.ts` | 视图间跳转辅助（today / prev / next / goto） |
| `src/stores/calendar.ts` | 日历 UI 状态（focusDate、showCompleted、heatmapOn 等） |
| `src/styles/calendar.css` | 月/周/年通用样式（单元格、任务块、热力图） |

### 4.3 可能需要的第三方日历组件

> **本节是用户重点关心的方向**。如果用户后续指定第三方组件，下面这些是主流候选：

| 库 | 优势 | 劣势 | 备注 |
|---|---|---|---|
| **FullCalendar** (`@fullcalendar/vue3`) | 工业级，支持月/周/年/日/列表多种视图，事件拖拽、Resize、热力图原生支持 | 包大（~200KB），样式定制需要覆盖大量 CSS | 最成熟方案 |
| **VueCal** | 纯 Vue，支持月/周/日、拖拽、事件编辑 | 体积中等，年视图无原生支持 | 适合周/月 |
| **Schedule-X** | 现代、轻量（~50KB），月/周/日 视图漂亮 | 年视图需自定义，热力图需自实现 | 适合周/月 |
| **自实现**（纯 Vue + CSS Grid） | 完全可控，零依赖 | 工作量大（每视图 200~300 行 CSS + 交互逻辑） | 长期维护成本低 |

**初步倾向**：先用 Schedule-X 或自实现跑通月/周，年视图自实现（4×3 网格很简单）。
**不倾向**：FullCalendar——太重，且样式风格与项目极简风不符。

> 用户表示"还要再找找日历组件让你用"——本节保留为待定项。

## 五、改动清单（实施时）

### 后端（视前端方案而定）

- **大概率不需要改后端**：`Task` 模型已有 `due_start_at` / `due_end_at`，前端按时间区间过滤即可
- **如果后端要支持**：新增 `tasks_in_range(start, end) -> Vec<Task>` command，返回区间内所有任务（含跨天任务）

### 前端

1. **路由**：router/index.ts 加 `/week`、`/month`、`/year` 三条
2. **store**：新建 `calendar.ts`，管理 focusDate / viewMode / heatmapOn 等 UI 状态
3. **AppLayout**：放宽 `showTaskSidebar` 包含 week/month/year；topbar 插入 `<ViewSwitcher />`
4. **新建 3 个视图组件** + **ViewSwitcher** + **CalendarPopover** + 共享 CSS
5. **快捷键**：在 `useShortcuts` 加 `L/W/M/Y` 切视图

### 估算工作量

| 视图 | 工作量估算 | 难度 |
|------|------------|------|
| ViewSwitcher（下拉） | 0.5 天 | 低 |
| 月视图 | 2~3 天 | 中（交互多：拖拽、点击空白、跳周） |
| 周视图 | 3~4 天 | 中高（时间轴 + 时间段任务 + 拖拽改时间） |
| 年视图 | 1~1.5 天 | 低（纯展示，热力图简单计算） |
| 共享 store / composable | 0.5 天 | 低 |
| 样式调优 | 1 天 | 中 |
| **合计** | **8~10 天** | |
